Org Babelによるorgファイル中でのソースコード実行。

init.elの設定は別ページで。

Org Babel によるpython実行

システムのpythonを使用 



#+BEGIN_SRC python :session system
def foo(x):
    if x > 0:o
        return x+1
    else:
        return x-1

return foo(5)
#+END_SRC

#+RESULTS:


#+BEGIN_SRC shell :session system
echo $VIRTUAL_ENV
which python
#+END_SRC

#+RESULTS:
|                                 |
|                                 |
| /home/myname/.pyenv/shims/python |

pipenvで作成したpython仮想環境を使用

あるいはpipenvで作成した環境を用いることもできる。 https://emacs.stackexchange.com/questions/24267/using-org-babel-to-enable-virtualenv のanswerに従うと以下のように設定することでpipenvで設定した仮想環境を使用できることを確認した。


#+BEGIN_SRC elisp :session venv
(pyvenv-workon "nogawafishSite-gnCTSypq")
#+END_SRC

#+RESULTS:


#+BEGIN_SRC shell :session venv
echo $VIRTUAL_ENV
which python
#+END_SRC

#+RESULTS:
| /home/myname/.virtualenvs/nogawafishSite-gnCTSypq            |
| /home/myname/.virtualenvs/nogawafishSite-gnCTSypq/bin/python |


#+BEGIN_SRC python :results output :session venv-scratch
import sys
print('\n'.join(sys.path))
#+END_SRC

#+RESULTS:

: /usr/lib/python3.8

#+BEGIN_SRC elisp :session venv
(pyvenv-deactivate)
